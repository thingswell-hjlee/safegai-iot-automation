# SOUL.md
## SafeGAI AI Fisheye Zone Safety
### 1차 산업안전 MVP 제품 원칙 v3.0

## 1. 제품 한 문장
AI 어안카메라 1대와 보조 AI 카메라 최대 3대로 하나의 위험공정을 감시하고, 범용 x86-64 Ubuntu 산업용 임베디드 PC 게이트웨이가 구역별 재실상태와 설비 운전상태를 결합하여 현장 경고, 설비 안전정지 요청, 관리자 알림과 안전 증빙을 제공하는 산업안전 제품이다.

## 2. 제품이 해결하는 단 하나의 문제
사람이 위험구역에 있거나 안전확인이 불가능한 상태에서 설비가 운전되거나 재가동되는 위험을 줄인다.

## 3. 첫 적용 공정
- 컨베이어 투입부·배출부
- 설비 정면·후면 접근구역
- 정비·청소 중 잔류자 확인이 필요한 단일 생산라인
- 하나의 현장 전체가 아니라 하나의 위험공정을 대상으로 한다.

## 4. 개발·시험 조직
- 개발자 1명: 제품설계, Gateway, AWS, Frontend, 통합, 릴리스 패키징
- 기능/UX 시험자 1명: 요구사항 인수기준, 사용자·운영자 기능, 회귀, 현장 사용성
- 성능/HIL 시험자 1명: 카메라·I/O·전원·통신·부하·장시간·테스트베드 시험
- 제품책임자는 범위, 안전정책, 출시 여부를 최종 결정한다.
- 개발자는 R3 안전변경을 단독 승인하거나 단독 출시하지 않는다.

## 5. 1차 제품 범위
### 포함
- AI 어안카메라 1대 필수, 전체 카메라 최대 4대
- 구역 최대 8개
- 구역별 재실, 인원, 진입, 이탈, 체류 이벤트 연동
- 설비 운전상태 및 재가동 요청 입력
- DI 8점, DO 8점의 절연형 외부 I/O
- 경광등, 사이렌, 저장 음성안내
- PLC 또는 Safety Relay에 안전정지 요청
- 사용자·운영자·유지보수자 모드
- 1·2·4분할 로컬 라이브뷰
- 이벤트 확인, 조치, 종료, 오감지 분류
- 정비 작업창과 재가동 전 잔류자 확인
- 로컬 이벤트·이미지·감사로그
- 최소 AWS 연계: 상태, 이벤트, 대표 이미지, 알림, 백업, 버전 조회
- 인터넷 장애 중 72시간 이상 독립운전

### 제외
- 게이트웨이 자체 영상 AI 추론
- 연속 영상 녹화와 VMS/NVR 대체
- 영상 트랜스코딩
- 얼굴인식과 개인 신원추적
- 범용 Rule/Activity 편집기
- AI 결과 단독에 의한 자동 재가동
- 클라우드 원격 설비제어
- MES/ERP/BMS 연동
- 예측 AI와 대규모 다중고객 SaaS

## 6. 역할 분리
### AI 카메라
- 사람 분류 및 영상 추론
- 구역 재실·인원·진입·이탈·체류 이벤트 생성
- 이벤트 이미지 또는 영상 URL 제공

### 범용 Ubuntu Gateway
- 카메라 제조사 이벤트 어댑터
- 재실 상태머신
- 설비·I/O 상태 결합
- 고정형 안전조건 판단
- 경고와 정지 요청
- 로컬 UI·이력·감사로그
- 오프라인 운전 및 AWS 비동기 동기화

### AWS
- 게이트웨이 상태와 버전
- 이벤트 메타데이터와 대표 이미지
- 외부 운영자 알림
- 원격 조회와 확인 동기화
- 안전제어 경로에는 참여하지 않는다.

## 7. 사용자 모드
- 현재 안전상태와 조치 안내
- 최대 4채널 영상
- 활성 경고와 최근 이벤트
- 허용된 이벤트 확인
- 설정·I/O 시험·유지보수 기능 접근 금지

## 8. 운영자 모드
- 위험 이벤트 확인·조치·종료·분류
- 정비 작업창 요청·종료
- 설비·카메라 상태와 기본 보고서
- 알림 대상과 비안전 운영 파라미터
- 안전 I/O 매핑과 핵심 안전조건 변경 금지

## 9. 유지보수자 모드
- 카메라 등록·통신시험·구역 매핑
- DI/DO 매핑과 개별 시험
- 로그·상태·저장공간·버전 확인
- 설정 백업·복원·업데이트·롤백
- 로컬 네트워크에서만 허용하며 변경은 감사로그에 기록한다.

## 10. 범용 게이트웨이 하드웨어 기준
- 하드웨어 등급: `SG-EPC-L1`
- 소프트웨어 프로파일: `ipc-lite-amd64-v1`
- 아키텍처: 표준 x86-64, UEFI 부팅
- CPU: 저전력 4코어, Intel Processor N100/N97/N150급 또는 동급 이상
- RAM: 8GB 기본, 16GB 선택
- 저장장치: 교체 가능한 M.2 SATA 또는 NVMe SSD 128GB 이상
- 네트워크: 1GbE RJ45 2포트 필수
- 현장 I/O: 절연형 Modbus TCP/RTU 8DI/8DO
- 전원: 12V DC 기본, 자동 전원복구; 9~36V 입력은 산업형 옵션
- 냉각: 팬리스 기본
- 운영체제: Ubuntu Server 24.04 LTS x86-64 기준
- 특정 제조사 커널, 보드 GPIO, 전용 부트로더, 전용 SDK에 의존하지 않는다.

## 11. 범용성 원칙
- 게이트웨이 소프트웨어는 표준 Ethernet, USB, RS485, Modbus, UEFI, SATA/NVMe만 사용한다.
- 현장 설비 신호는 외부 절연형 I/O를 통해 연결하며 PC GPIO를 사용하지 않는다.
- 동일 소프트웨어 패키지가 서로 다른 제조사 PC 2종에서 동작해야 한다.
- 제품 출시는 모델명보다 성능등급과 필수 인터페이스를 기준으로 승인한다.
- 특정 PC 단종 시 소프트웨어 수정 없이 동급 PC로 교체할 수 있어야 한다.

## 12. 절대 안전원칙
- 본 제품은 AI 기반 보조 안전·관제 제품이며 인증된 Safety PLC, Safety Relay, 비상정지와 방호장치를 대체하지 않는다.
- 일반 DO로 기계 동력을 직접 차단하지 않는다.
- 정지 요청은 기존 PLC, Safety Relay 또는 승인된 안전회로에 전달한다.
- 카메라 장애·가림·이벤트 지연은 `VACANT`가 아니라 `UNKNOWN` 또는 `STALE`이다.
- `UNKNOWN`과 `STALE`에서는 재가동 허용을 차단한다.
- AI 비재실만으로 고위험 설비를 자동 재가동하지 않는다.
- 비상정지와 기존 안전장치는 SafeGAI보다 항상 우선한다.
- AWS 장애가 현장 경고·정지 요청에 영향을 주지 않는다.

## 13. 성능 목표
- 이벤트 수신 후 경고 시작: p95 1초 이내
- DO 정지요청 시작: p95 500ms 이내
- 로컬 화면 이벤트 반영: p95 1초 이내
- 카메라 장애 판정: 30초 이내
- 4채널 H.264 보조스트림 표시
- 기준 PC에서 평균 CPU 40% 이하, p95 70% 이하
- 정상상태 애플리케이션 메모리 3GB 이하
- 전원복구 후 핵심기능 3분 이내
- 72시간 클라우드 단절 시험 통과
- 20회 전원 반복 시험 통과
- 30일 현장 연속운전

## 14. 출시 원칙
- 8주에 MVP Release Candidate를 만들고 2주 현장 안정화 후 Pilot Release한다.
- 기준 카메라는 2개 제조사 이하로 제한한다.
- 범용 게이트웨이 PC는 서로 다른 제조사 2종을 적합성 시험한다.
- 기능 추가보다 오감지 분류, 장애처리, 설치성, 이력증빙을 우선한다.
- 고객별 요구는 설정과 어댑터로 처리하고 고객별 소스분기를 만들지 않는다.
